Actividad teórico-práctica sobre formularios

La etiqueta form

La etiqueta <form> se usa para crear un formulario HTML donde el usuario ingresará información.

El elemento <form> puede contener uno o más de los siguientes elementos de formulario:

  • <input>
  • <textarea>
  • <button>
  • <selet>
  • <option>
  • <optgroup>
  • <fieldset>
  • <label>
  • <output>

Algunos de los atributos que pueden utilizarse en <form> son los siguientes:

  • action

    Su value es la URL a donde se destinan los datos.
    Especifica a dónde enviar los datos del formulario cuando se envía un formulario.

  • autocomplete

    Su value puede ser on u off.
    Especifica si un formulario debe tener el autocompletado activado o desactivado.

  • enctype

    Su value puede ser application/x-www-form-urlencoded, multipart/form-data o text/plain.
    Especifica cómo deben codificarse los datos del formulario al enviarlos al servidor (solo para method="post").

  • method

    Su valuepuede ser get o post.
    Especifica el método HTTP que se usará al enviar los datos del formulario.

  • name

    Su value es algún texto que detalle el nombre que queremos darle.
    Especifica el nombre del formulario.

  • novalidate

    Su value es novalidate.
    Especifica que el formulario no debe validarse cuando se envía.

  • target

    Su value puede ser _blank, _self, _parent o top.
    Especifica dónde mostrar la respuesta que se recibe después de enviar el formulario.


Campos de texto: input

Los campos de entrada de los formularios se definen mediante el tag <input> y sus diferentes valores. Estos valores nos permiten utilizar varios tipos de cajas y de formas."


Para camnbiar el formato de un campo de texto, hay que hacer uso del atributo type, y los formatos de datos disponibles son los siguientes:

  • <input type="text">

    Este tipo de campos se usan para albergar informaciones cortas y concretas.

  • <input type="number">

    El tipo number es usado para permitir al usuario ingresar un número. Incluye una validación para rechazar entradas no numéricas. El navegador puede optar por proveer flechas de pasos para permitir al usuario incrementar y decrementar el valor usando su ratón o simplemente pulsando con la punta del dedo.

  • <input type="email">

    Campo para introducir una dirección de correo electrónico. Las pseudoclases :valid e :invalid son aplicadas según corresponda.

  • <input type="url">

    Este tipo se usa para los campos de entrada que deben contener una dirección URL.
    Dependiendo del soporte del navegador, el campo de URL puede validarse automáticamente cuando se envía.
    Algunos teléfonos inteligentes reconocen el tipo de url y agregan ".com" al teclado para que coincida con la entrada de url.

  • <input type="tel">

    Se usa para campos de entrada que deben contener un número de teléfono. Puede combinarse con el atributo pattern para especificar una expresión regular para verificar el valor de entrada.

  • <input type="checkbox">

    Este campo sirve para crear casillas de verificación que le permiten al usuario seleccionar cero o más opciones dentro un número limitado de opciones.

  • <input type="radio">

    Son botones de opción que le permiten al usuario seleccionar solo una opción de un número limitado de opciones.

  • <input type="date">

    Se usa para campos de entrada que deben contener una fecha.
    Según el soporte del navegador, puede aparecer un selector de fecha en el campo de entrada.

  • <input type="time">

    Permite al usuario seleccionar una hora (sin zona horaria).
    Dependiendo del soporte del navegador, puede aparecer un selector de tiempo en el campo de entrada.

  • <input type="datetime-local">

    especifica un campo de entrada de fecha y hora, sin zona horaria.
    Según el soporte del navegador, puede aparecer un selector de fecha en el campo de entrada.

  • <input type="password">

    Define un campo de contraseña. Los caracteres en un campo de contraseña están enmascarados (se muestran como asteriscos o círculos).

  • <input type="search">

    Se usa para campos de búsqueda (un campo de búsqueda se comporta como un campo de texto normal).

  • <input type="submit">

    Define un botón para enviar datos de formulario a un controlador de formulario.
    El controlador de formulario suele ser una página de servidor con un script para procesar datos de entrada.
    El controlador de formulario se especifica en el atributo de acción del formulario.

  • <input type="reset">

    Define un botón de reinicio que restablecerá todos los valores del formulario a sus valores predeterminados.

  • <input type="color">

    Se usa para campos de entrada que deben contener un color.
    Dependiendo de la compatibilidad del navegador, puede aparecer un selector de color en el campo de entrada.

  • <input type="file">

    Define un campo de selección de archivo y un botón "Examinar" para cargar archivos.

  • <input type="hidden">

    Define un campo de entrada oculto.
    Un campo oculto permite a los desarrolladores web incluir datos que los usuarios no pueden ver ni modificar cuando se envía un formulario.
    Un campo oculto a menudo almacena el registro de la base de datos que debe actualizarse cuando se envía el formulario.

  • <input type="image">

    Define una imagen como un botón de enviar.
    La ruta a la imagen se especifica en el atributo src.

  • <input type="month">

    Permite al usuario seleccionar un mes y año.
    Según el soporte del navegador, puede aparecer un selector de fecha en el campo de entrada.

  • <input type="range">

    Define un control para ingresar un número cuyo valor exacto no es importante (como un control deslizante). El rango predeterminado es de 0 a 100. Sin embargo, puede establecer restricciones sobre qué números se aceptan con los atributos min, max y step.

  • <input type="week">

    Permite al usuario seleccionar una semana y un año.
    Según el soporte del navegador, puede aparecer un selector de fecha en el campo de entrada.


Aquí hay una lista de algunas restricciones de entrada comunes:

  • size: mediante el atributo size podemos definir el tamaño de la caja. Es decir, la apariencia de la misma. Si el texto que vamos a escribir no cabe en la caja, éste se desplazará pero sin que la caja varíe su tamaño. El texto irá desapareciendo por la izquierda.

  • minlength: este atributo nos sirve para definir la cantidad mínima de letras que se pueden escribir en la caja. No se podrán escribir en la caja menos caracteres que los indicados por el minlength.

  • maxlength: este atributo nos sirve para definir la cantidad máxima de letras que se pueden escribir en la caja. No se podrán escribir en la caja más caracteres que los indicados por el maxlength.

  • value: define si queremos que en la ventana haya un texto ya preescrito. Este texto puede ayudar al usuario a saber cúal es el tipo de dato que te piden en ese campo.

  • name: para ponerle un nombre al campo. Esto resulta muy útil a la hora de gestionar la información que nos llegue.

  • required: este atributo especifica que el usuario debe llenar obligatoriamente el campo antes de poder enviarse el formulario. En caso de no llenarse, aparecerá un cartel advirtiendo al usuario que se precisa que se llene el campo para poder efectivamente enviar la información.
    (Este atributo no necesita escribirse en comillas, con solo escribir required dentro de la etiqueta de input ya es suficiente.)

  • checked: especifica que se debe preseleccionar un campo de entrada cuando se carga la página (para type="checkbox" o type="radio").

  • disabled: especifica que un campo de entrada debe estar deshabilitado.

  • max: especifica el valor máximo para un campo de entrada.

  • min: especifica el valor mínimo para un campo de entrada.

  • pattern: especifica una expresión regular para verificar el valor de entrada.

  • readonly: especifica una expresión regular para verificar el valor de entrada.

  • step: especifica los intervalos de números legales para un campo de entrada.

  • placeholder: se puede añadir un indicación para el usuario que defina que se debe insertar en el campo.


Campos de texto largo: textarea

La etiqueta <textarea> define un control de entrada de texto de varias líneas.
Un área de texto puede contener un número ilimitado de caracteres, y el texto se representa en una fuente de ancho fijo (generalmente Courier).
El tamaño de un área de texto puede especificarse mediante los atributos de cols y rows, o incluso mejor; a través de las propiedades de width y height de CSS.
Consejo: ¡Agregá siempre la etiqueta <label> para hacer uso de buenas prácticas de accesibilidad!


La etiqueta select

El elemento <select> se usa para crear una lista desplegable.
Las etiquetas <option> dentro del elemento <select> definen las opciones disponibles en la lista.


La etiqueta button

La etiqueta <button> define un botón en el que se puede hacer clic.
Dentro de un elemento <button> se puede poner contenido como texto o imágenes. Esta es la diferencia entre este elemento y los botones creados con el elemento <input>.
A tener en cuenta: si usás el elemento <button> en un formulario HTML, diferentes navegadores pueden enviar valores diferentes. Usá mejor <input> para crear botones en un formulario HTML.


La etiqueta label

El elemento <label> define una etiqueta para muchos elementos, como <input>, <textarea>, <select>, <meter> y <progress>.
